# Vibe Crafting Blog Post

Blog-post-as-repo about "Vibe Crafting" — a deliberate process for building production-quality apps with AI. Each section is drafted as its own file, then assembled into a final README.

## Key Files

- `specs/planning/phase_instructions.md` — How to write a "phase" of the blog. Start here.
- `specs/planning/implementation_plan.md` — Build checklist (track progress here)
- `specs/planning/outline.md` — Full outline with structure and bullet points for every section
- `specs/planning/needed_prompts.md` — Prompt files that need to be collected and linked

## Project Structure

- `specs/sections/` — Individual section drafts. These get concatenated into the final `README.md`.
- `specs/project_overview/` — Raw notes and ideas (source material, not published)
- `specs/planning/` — Outline, implementation plan, phase instructions
- `prompts/` — Real prompt files from the project being described. Referenced and linked from sections.

## Task

This is a writing project, not a coding project. You are drafting blog sections as markdown files. Follow the phase instructions for the full workflow.

## Style

- Conversational, honest, technical but accessible. First person.
- Not preachy or salesy. Show, don't tell — use concrete examples over generic claims.
- Sections should read well standalone and as part of the assembled whole.
- Link to example prompt files in `prompts/` where the it calls for it.
